git log
显示从最近到最远的提交日志

版本回退：
HEAD：表示当前版本，当前分支
HEAD^：上一个版本
HEAD^^：上上一个版本
HEAD~100：往上100个版本

git checkout – file丢弃工作区的修改：

  • 一种是文件自修改后还没有git add添加，现在，撤销修改就回到和版本库一模一样的状态；
  • 一种是文件已经git add，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

git checkout其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。
