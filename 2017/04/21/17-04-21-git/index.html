git使用

参考自廖雪峰的官方网站:
http://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000/

添加文件到本地暂存区:

  1. 添加单个文件

  2. 添加多个文件[用空格区分]

git commit把暂存区的所有内容提交到当前分支:

  1. 回车进入vim输入说明文本，按键盘上[esc]输入:wq 保存退出vim

git diff file
//点击键盘上的[q]退出diff

git log //显示的更为详细
git log –pretty=oneline //显示版本号和版本说明

git reset –hard HEAD^ //回退到上一个版本
git reflog //查看git的每一步动作
git reset –hard 未来版本id
git reset HEAD file //可以把暂存区的修改撤销掉（unstage），重新放回工作区

git checkout –

//要关联一个远程库，使用命令
git remote add origin git@server-name:path/repo-name.git

//第一次推送master分支的所有内容到远程库
git push -u origin master

//推送最新修改到远程库
git push origin master

git clone git@server-name:path/repo-name.git

git checkout -b <name>


git branch <name>


git checkout <name>

git branch

//dev分支会被合并到当前分支中
git merge [要被合并的分支名称]

git branch -d

git branch -D

//查看分支合并图
git log –graph
git log –graph –pretty=oneline –abbrev-commit

git merge –no-ff -m “本次合并要创建一个commit的描述文本”

//转移正在开发的内容到stash
git stash

//查看stash
git stash list

//将stash中的内容恢复，并将stash内容删除
git stash pop

//将stash中的内容恢复
git stash apply

//删除stash中的内容
git stash drop

git stash apply stash@{0}

git tag v1.0[标签名称]

git tag v0.9 6224937[commit id]

git tag -a v0.1[标签名称] -m “version 0.1 released 标签说明” 3628164[commit id]

git tag


git show <tagname>


git tag -d <tag-name>


git push origin :refs/tags/tagname

git push origin

//推送全部尚未推送到远程
git push origin –tags

//显示颜色
git config –global color.ui true
//别名
git config –global alias.st status

git fetch origin master
git merge origin/master
```
git pull：从远程仓库获取最新版本，自动merge到本地仓库
